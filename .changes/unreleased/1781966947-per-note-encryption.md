title: Per-note, per-attachment encryption
doc: per-note-encryption

At-rest encryption now seals each note as its own compressed, opaquely-named encrypted file and each attachment as its own encrypted blob — never folded together — so a note opens by downloading only its own attachments, on demand, and a green lock fills in note-by-note as a paced background migration seals everything without flooding the cloud; every conversion is atomic, so nothing can be lost.
